# ADR_03: Roadmap de Desarrollo y Estrategia de Implementación Fullstack

**Fecha**: 23 de Abril, 2026  
**Estado**: Aceptado  
**Proyecto**: AUTH_entic

---

## 1. Contexto y Estrategia de Trabajo

Este documento define la estrategia de implementación para la aplicación fullstack **AUTH_entic**, priorizando el aprendizaje continuo, la integración fluida entre capas y el uso eficiente de herramientas de inteligencia artificial.

- **Vertical Slices (Rebanadas Verticales)**: Se desarrollarán funcionalidades completas que atraviesen todas las capas (Front-End -> API -> Base de Datos) en lugar de construir el proyecto por capas horizontales aisladas.
- **Comando de IA**: Se utilizarán herramientas modernas (como V0, Lovable o similares) para el prototipado rápido de UI con **shadcn/ui**, delegando a la IA la generación de componentes visuales, mientras el desarrollador asume el control total de la lógica de negocio, arquitectura e integración.
- **Supervisión Senior AI**: La asistencia de IA actuará como supervisor, revisor de código y guía arquitectónica, interviniendo directamente en la escritura de código solo bajo solicitud explícita.

## 3. Fases de Implementación (Roadmap)

### Fase 1: Inicialización del Frontend (Punto de Partida)

- **Configuración**: Inicializar proyecto React con Vite en el directorio `apps/frontend/`.
- **Infraestructura UI**: Configurar Tailwind CSS e inicializar los componentes base de **shadcn/ui**.
- **Routing**: Implementar la estructura base y el proveedor de rutas con **TanStack Router** (`/`, `/login`, `/register`, `/dashboard`).

### Fase 2: Slice #1 - Registro de Usuarios (Sign Up)

- **Frontend**: Crear formulario de registro con validación estricta en el cliente usando Zod y React Hook Form.
- **Integración**: Conectar el formulario con el endpoint `POST /users` ya existente en el backend.
- **Objetivo**: Poblar la base de datos real, verificar la conectividad del monorepo y validar que el hash de contraseñas con **bcrypt** funcione correctamente de extremo a extremo.

### Fase 3: Slice #2 - Autenticación (Login & JWT)

- **Backend**: Implementar el módulo de autenticación (AuthModule), servicio de validación de credenciales y generación de **JWT**.
- **Frontend**: Crear página de Login y establecer la estrategia de persistencia del token (por definir: Cookies HttpOnly o LocalStorage).
- **Gestión de Estado**: Configurar el `AuthContext` o utilizar hooks de **TanStack Query** para manejar el estado global de la sesión del usuario.

### Fase 4: Slice #3 - Dashboard y Autorización (Mantenedores)

- **Layout**: Construir la estructura principal del Dashboard (Sidebar/Header) asegurando navegación protegida.
- **Backend**: Implementar **Guards** y decoradores en NestJS para proteger rutas basándose en roles de usuario (`admin` vs `user`).
- **Funcionalidad**: Construir el mantenedor de usuarios (CRUD) consumiendo endpoints que requieran autorización.

### Fase 5: Slice #4 - Sistema de Posts (Lógica de Negocio Central)

- **Backend**: Crear entidad `Post`, controladores y servicios asociados. Implementar lógica de filtrado estricta: los usuarios regulares solo pueden interactuar y ver posts de otros usuarios regulares; los administradores tienen acceso global o segmentado según su rol.
- **Frontend**: Construir el "Post Board" utilizando **TanStack Query** para optimizar la caché, las mutaciones optimistas y los estados de carga.

---

## 4. Decisiones Técnicas Clave Derivadas

- **Decisión sobre el punto de inicio**: Se decide comenzar por el **Registro (Sign Up)** en lugar del Login. Esto aprovecha el trabajo previo en el endpoint de creación de usuarios del backend y permite validar inmediatamente la conexión Front-Back.
- **Gestión de peticiones HTTP**: Se adopta **TanStack Query** como estándar para todas las peticiones asíncronas en el frontend, garantizando una correcta sincronización de estado del servidor y mitigando la complejidad del manejo de datos asíncronos en React puro.
