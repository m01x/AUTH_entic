# ADR: Implementación de Endpoints CRUD en UserController y UserService — Prefijo API Global

## Contexto
Para estandarizar las operaciones básicas sobre la entidad `User` y sentar las bases de la API, se requiere la implementación de los endpoints CRUD faltantes en `UserController`. A su vez, `UserService` debe contener la lógica inicial (stub) para estas operaciones, asegurando que el código compile y los endpoints sean visibles, aunque su funcionalidad completa se implementará en el futuro.

Adicionalmente, se establece la necesidad de un prefijo global para toda la API (`/api`) para mantener una nomenclatura consistente y evitar colisiones de rutas.

## Decisiones Técnicas y Configuración Crítica

### 1. Nuevos endpoints en UserController
Se han añadido los siguientes endpoints al controlador `users.controller.ts`, siguiendo las convenciones de NestJS para operaciones RESTful:

- **`GET /users/:id`**: Mapeado al método `getById(id)`. Delega la lógica al método `findById(id)` del servicio.
- **`DELETE /users/:id`**: Mapeado al método `delete(id)`. Delega la lógica al método `remove(id)` del servicio.
- **`PATCH /users/:id`**: Mapeado al método `update(id, body)`. Delega la lógica al método `update(id, body)` del servicio.

### 2. Lógica stub en UserService
Para permitir que los endpoints del controlador sean funcionales sin tener la lógica de negocio completa, se implementaron los siguientes métodos en `users.service.ts` como "stubs":

- `findById(_id: string)`
- `remove(_id: string)`
- `update(_id: string, _updateUserDto: UpdateUserDto)`

Todos estos métodos lanzan una `NotImplementedException` con el mensaje `"no implementado"`. Esto sirve como un marcador claro para el desarrollo futuro y asegura que cualquier intento de usar estos endpoints arroje un error 501 Not Implemented, comunicando explícitamente que la funcionalidad aún no está disponible.

### 3. Prefijo API global
Desde este momento se agrega el prefijo `api` a todo el proyecto. Todas las rutas de la API estarán anidadas bajo `/api`. Por ejemplo, el endpoint para obtener todos los usuarios será `GET /api/users`.

## Consideraciones y Flujo de Trabajo (Troubleshooting)
- **Desarrollo Futuro**: Para implementar la funcionalidad completa, se debe reemplazar el `throw new NotImplementedException(...)` en cada método del servicio con la lógica de base de datos correspondiente (consultas, actualizaciones, eliminaciones).
- **Diagnóstico**: Si se recibe un error `501 Not Implemented` al consumir estos endpoints, significa que la lógica de negocio en el servicio aún no ha sido desarrollada. No es un error de ruteo o de controlador.
